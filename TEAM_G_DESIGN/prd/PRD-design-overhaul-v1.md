## 1. 개요

### 문제 정의

autoplan 6-voice 리뷰에서 디자인 전 영역 합의된 문제:

- **Social/Viral 3/10**: 카카오톡 JS SDK 미연동, 공유가 플레인 URL
- **Retention 4/10**: 대시보드에 데일리 콘텐츠 훅 없음, 푸시 없음
- **FTUX 6/10**: 인증벽 전 가치 0, 가입 전 프리뷰 없음
- **Conversion 5/10**: 3회 하드월 → 결정 마비 (3 CTA), 블러 프리미엄 없음
- **Missing States D**: 70+ 페이지에 에러 바운더리 4개, 죽은 페이지 존재
- **접근성 C**: 골드 텍스트 WCAG AA 미달, ARIA 부족

### 목표

| 지표               | 현재 | 목표 | 측정                    |
| ------------------ | ---- | ---- | ----------------------- |
| Social/Viral Score | 3/10 | 8/10 | 카카오 리치 카드 공유율 |
| Retention Score    | 4/10 | 8/10 | D1/D7/D30 리텐션        |
| FTUX Score         | 6/10 | 9/10 | 가입 전환율             |
| Conversion Score   | 5/10 | 8/10 | Free→Paid 전환율        |
| Error Coverage     | 4개  | 전체 | 에러 바운더리 커버리지  |
| WCAG AA 준수       | 부분 | 전체 | Lighthouse 접근성 점수  |

### 성공 지표 KPI

- 카카오톡 공유 후 유입률: 0% → 15%+
- D7 리텐션: 측정불가 → 30%+
- 가입 전환율: 측정불가 → 20%+
- Free→Paid 전환율: 측정불가 → 5%+

## 3. 기능 요구사항

### MVP — Phase 1: 리텐션 + 바이럴 (1주)

#### F-01: 대시보드 데일리 포춘 카드

- 위치: `/protected/analysis` 최상단 (MasterpieceSection 위)
- 기존 `DailyFortuneView` 컴포넌트 재활용
- 오늘의 운세 요약 (1-2줄) + "자세히 보기" 링크
- 연속 접속 스트릭 카운터 (출석 시스템 연동)
- 미로딩 시 스켈레톤 UI
- **기존 코드**: `components/analysis/daily-fortune-view.tsx`, `app/actions/fortune/daily.ts`

#### F-02: 카카오 JS SDK 연동

- 현재: Web Share API 폴백 (플레인 URL)
- 변경: `Kakao.Link.sendDefault()` 리치 미리보기 카드
  - 이미지: OG 카드 (기존 `/api/og` 엔드포인트 활용)
  - 제목: "청담해화당 사주분석 결과"
  - 설명: 분석 요약 1줄
  - 버튼: "내 운명 보기" (앱 링크)
- 결과 페이지 하단에 공유 CTA 배치 (감정 피크 활용)
- **기존 코드**: `components/share/viral-share-button.tsx` (수정)
- **새 파일**: `lib/kakao-sdk.ts` (SDK 초기화)

#### F-03: 인증벽 전 미니 리딩

- 랜딩 페이지에 생년월일 입력 폼 추가 (년/월/일 3필드)
- 입력 즉시: 띠(동물) + 오행(목화토금수) + 일간 성격 한 줄
- 서버 액션 없이 클라이언트 계산 (lib/domain/saju/saju.ts 활용)
- 결과 하단: "전체 사주풀이 보기 → 회원가입" CTA
- 기존 `ReviewMarquee` + `LiveMemberCounter` 랜딩에 렌더링 (현재 빌드됨 but 미사용)
- **기존 코드**: `lib/domain/saju/saju.ts` (getSajuData), `components/landing/`

### Phase 2: 전환 + 안정성 (2주)

#### F-04: 블러 프리미엄 콘텐츠

- 무료 결과: 1-2 섹션 공개, 나머지 CSS `blur(8px)` + 오버레이
- 오버레이: "전체 분석 잠금 해제" 단일 CTA
- 현재 PaywallModal(3 CTA) → 단일 CTA + 부가 링크로 간소화
- **기존 코드**: `components/shared/paywall-modal.tsx` (리팩토링)

#### F-05: 에러 바운더리 전역화

- 모든 `dynamic()` import에 error fallback 추가
- 라우트 그룹별 error.tsx 생성:
  - `app/protected/analysis/error.tsx`
  - `app/protected/studio/error.tsx`
  - `app/protected/fortune/error.tsx`
  - `app/protected/membership/error.tsx`
- 에러 유형별 UI: 네트워크/인증/서버/알수없음
- "다시 시도" + "홈으로" 듀얼 CTA

#### F-06: 페이월 단일 CTA 리디자인

- 현재: "멤버십 가입" + "복채 충전" + "1회 계속" (3개)
- 변경: "잠금 해제" 프라이머리 CTA + "다른 방법" 텍스트 링크
- 분석별 복채 비용 배지를 대시보드 카드에 표시

### Phase 3: 디자인 시스템 정비 (3주)

#### F-07: 시맨틱 컬러 토큰 확장

- 추가: error(#EF4444), success(#22C55E), warning(#F59E0B), info(#3B82F6)
- CSS 변수 기반 다크모드 전환 준비
- opacity 변형: `/10`, `/20`, `/50`, `/80`

#### F-08: 접근성 개선

- 골드 텍스트 대비 WCAG AA 검증 + 수정
  - `#C8B273` on `#0D0D0D` = 7.8:1 (OK)
  - `#C8B273` on `#1A1917` = 6.2:1 (OK)
  - `text-ink-light/50` (#ffffff80) on dark = 미달 → 최소 `/70` 이상
- 캐러셀 ARIA: `role="region"`, `aria-live="polite"`
- `prefers-reduced-motion` 지원
- 키보드 내비게이션: 포커스 트랩 (모달), 탭 순서

#### F-09: 타이포그래피 스케일

- h1~h6, body, caption, overline 정의
- line-height, letter-spacing 표준화
- 한글 최적화: word-break, 줄바꿈 규칙
