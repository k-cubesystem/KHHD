# 🐛 버그 수정 워크플로우

## 개요
프로덕션 또는 개발 환경의 버그를 빠르게 찾고 수정하는 프로세스

## 예상 시간
30분 - 1시간 (버그 심각도에 따라)

## 심각도 분류

### Critical (즉시 처리)
- 서비스 중단
- 데이터 손실
- 보안 취약점

### High (24시간 내)
- 주요 기능 장애
- 사용자 경험 심각한 저하

### Medium (1주일 내)
- 부분적 기능 오류
- UI 버그

### Low (여유시간)
- 사소한 표시 오류
- 개선 사항

---

## 에이전트 실행 순서

### 1단계: 버그 트리아지 (5분)
**담당**: CLAUDE

**작업**:
- 버그 심각도 판단
- 필요 에이전트 선택
- 우선순위 결정

---

### 2단계: 버그 재현 및 원인 분석 (15-30분)
**담당**: SHERLOCK

**작업**:
```typescript
// 1. 재현 단계 기록
// 2. 에러 로그 확인
// 3. Network 탭 확인
// 4. React DevTools로 상태 추적
// 5. 근본 원인 식별

[버그 리포트]
- 재현 방법: ...
- 에러 메시지: ...
- 근본 원인: ...
- 영향 범위: ...
```

---

### 3단계: 버그 수정 (20-30분)
**담당**: FE_LOGIC / BE_SYSTEM (버그 위치에 따라)

**작업**:
- 근본 원인 해결
- 관련 엣지 케이스 함께 수정
- 타입 안정성 보장

---

### 4단계: 회귀 테스트 (10분)
**담당**: SHERLOCK

**작업**:
- 수정 사항 검증
- 기존 기능 영향 확인
- 엣지 케이스 재테스트

---

### 5단계: 핫픽스 배포
**담당**: BOOSTER

**작업**:
```bash
git checkout -b hotfix/bug-name
git add .
git commit -m "fix: Bug description"
git push origin hotfix/bug-name
# PR 생성 → 머지 → 즉시 배포
```

## 예시: 무한 로딩 버그

**증상**: 사주 분석 페이지 로딩 무한 반복

**SHERLOCK 분석**:
```typescript
// 원인: useEffect 의존성 배열 문제
useEffect(() => {
  fetchData();
}, [data]); // ❌ 객체 참조 변경으로 매번 재실행

// 해결
useEffect(() => {
  fetchData();
}, [data.id]); // ✅ primitive 값만 의존성
```

**소요 시간**: 30분
