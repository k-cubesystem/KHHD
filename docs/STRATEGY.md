# 해화당 AI 2.0 확장 개발 전략 (STRATEGY)

## 1. 자율주행 오케스트레이션 전략
- **Gemini 3 Pro (오케스트레이터)**: 전체 설계 및 비즈니스 로직, 문서화 담당.
- **Claude Code (프론트엔드 장인)**: 복잡한 UI 개발, 애니메이션 구현, FE 디버깅 하달.
- **Watchdog**: 3회 연속 디버깅 실패 시 대체 전략(Strategy B) 수립 후 보고.

## 2. 기술적 핵심 과제 및 해결 방안

### A. OAuth 리다이렉트 이슈 (최우선)
- **진단**: 하이드레이션 불일치와 Supabase SSR 설정 간의 충돌.
- **해결**: `isMounted` 훅을 통한 안정적인 클라이언트 렌더링 보장 및 `e.preventDefault()`를 통한 기본 동작 제어.
- **협업**: Claude를 통해 `/auth/login` 페이지의 UX를 대폭 강화하고 소셜 로그인 버튼을 파편화된 곳이 아닌 중앙 집중 시스템으로 개편.

### B. 멀티모달 데이터 정합성 (天-地-人)
- **데이터 흐름**: 사주 데이터(텍스트) + 관상/손금(이미지) + 풍수(이미지/지도)를 하나의 컨텍스트로 묶어 Gemini 1.5 Pro에 전달.
- **프롬프트 엔진**: '해화당 마스터'의 유려한 문체를 유지하면서, 각 데이터 간의 상관관계를 분석하는 고도화된 프롬프트 체인 구축.

### C. 서비스 생태계 구축 (Social & Loop)
- **카카오 연동**: 사용자 재방문을 유도하는 '행운의 알림' 로직을 Supabase Edge Functions로 자동화.
- **공유 가능성**: 분석 결과를 유려한 이미지 카드로 생성해 인스타그램/카카오톡 공유 유도.

## 3. Phase 1 실천 과제 (Immediate Action)
1. **OAuth 버튼 안정화**: 모든 페이지에서 통합 소셜 로그인 컴포넌트가 정상 동작하도록 수정.
2. **디자인 시스템 확립**: `tailwind.config.ts` 및 `globals.css`에 골드/블랙 테마 및 유리 질감 유틸리티 강화.
3. **대시보드 레이아웃**: Claude를 호출하여 `protected/page.tsx`를 프리미엄 대시보드로 전면 재구성.
