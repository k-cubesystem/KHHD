# 해화당(海花堂) 서비스 고도화 및 시스템 통합 설계서 (Revised)

## 1. 개요 및 추진 방향
본 문서는 기존에 구축된 사주, 관상, 풍수 기능을 전면 재개발하지 않고, **사용자 경험(Flow)의 연결성**을 강화하고 **모든 기능의 관리 통제권**을 확보하는 데 목적이 있습니다. 점진적인 리팩토링을 통해 시스템을 고도화합니다.

---

## 2. 사용자 경험(UX) 및 기능 흐름 재설계
단순 페이지 이동이 아닌, **'데이터(Context)를 품은 이동'**으로 설계를 변경합니다.

### 2.1. 인연 중심의 원스톱 실행
*   **AS-IS**: 메뉴 이동 -> 대상 선택 -> 분석 실행
*   **TO-BE**: 대상 선택(프로필/인명부) -> **"이 사람으로 무엇을 보시겠습니까?"** 메뉴 팝업 -> 즉시 분석 실행
    *   **기술적 구현**: URL Query Parameter 활용 (예: `/analysis/saju?targetId=123&mode=wealth`)
    *   **데이터 연동**: `family_members` 테이블의 ID를 각 기능 페이지가 받아와 자동 로딩.

### 2.2. 상황별 맞춤 네비게이션 (Contextual Action)
*   사용자 대시보드에 고정된 메뉴 대신, 사용자의 상황(시간, 이전 기록)에 따른 **동적 액션 버튼** 배치.
    *   아침: "오늘의 기운 확인하기" (오늘의 운세)
    *   저녁: "내일의 행운 준비하기" (풍수/방위)
    *   특정인 조회 후: "이 사람과의 관계 더 알아보기" (궁합)

---

## 3. 관리자(Admin) 제어 시스템 설계
모든 기능은 **'Admin First'** 원칙하에 설계됩니다. 하드코딩을 최소화하고 DB 설정을 따릅니다.

### 3.1. 기능 제어 (Feature Flags)
코드를 배포하지 않고도 특정 기능을 켜고 끌 수 있는 시스템을 구축합니다.
*   **테이블 설계**: `system_config` 또는 `feature_flags`
    *   `key`: 기능 식별자 (예: `saju_today`, `face_analysis`, `payment_danal`)
    *   `is_active`: 활성화 여부 (boolean)
    *   `access_level`: 접근 권한 (all, beta_user, admin_only)
    *   `maintenance_msg`: 비활성 시 노출할 점검 메시지

### 3.2. 관리자 대시보드 UI 확장 (Service Control Center)
관리자 페이지(`/admin`)에 **[서비스 기능 제어]** 탭을 신설하여 미리 자리를 잡아둡니다. (기능 구현은 추후 진행)
*   **주요 관리 항목**:
    *   **메뉴 노출 관리**: 대시보드 메인 버튼 순서 및 노출 여부 제어.
    *   **가격 정책 관리**: 각 분석 기능별 코인(부적) 소모량 실시간 수정.
    *   **문구/프롬프트 관리**: 분석 결과에 나가는 멘트나 AI 프롬프트 템플릿 수정.

---

## 4. 단계별 실행 계획 (Roadmap)

### Phase 1: 기반 구조 마련 (Foundation)
1.  **DB 스키마 점검**: `feature_flags` 테이블 생성 및 `ai_prompts` 테이블 연동 확인.
2.  **어드민 페이지 UI 확장**: '서비스 제어' 탭 껍데기(Scaffold) 생성.
3.  **공통 유틸리티 작성**: `checkFeatureStatus(key)`와 같은 헬퍼 함수 작성 (프론트/백엔드 공용).

### Phase 2: 인연 관리(Relationship) 기능 연결
1.  기존 `family_members` 데이터를 프로필 페이지와 연동 강화.
2.  대상 선택 시 뜨는 **'액션 시트(Action Sheet)'** UI 컴포넌트 개발 (여기서 사주/관상/풍수 버튼 노출).
3.  각 기능 페이지(`saju/today`, `saju/compatibility` 등)가 `targetId` 파라미터를 받아 자동으로 분석을 시작하도록 `useEffect` 로직 수정.

### Phase 3: 관리자 기능 연동
1.  주요 버튼(메인 대시보드 진입점)에 `checkFeatureStatus` 가드 적용.
2.  관리자 페이지에서 스위치(Toggle) 조작 시 실제 앱 버튼이 잠기거나 사라지는지 테스트.

---

## 5. 결론
이 설계는 기존에 개발된 사주 엔진과 UI 컴포넌트를 **100% 재활용**합니다. 다만, 사용자가 기능을 찾아가는 **'길(Path)'**을 더 편하게 뚫어주고, 그 길을 운영자가 **'신호등(Admin)'**으로 통제할 수 있게 만드는 작업입니다.
